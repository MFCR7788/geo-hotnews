## 三、支付方案

### 3.1 支付渠道

**虎皮椒 V3**（聚合微信支付 + 支付宝）
- ✅ 个人可用，无需企业资质
- ✅ 接入最简，比原生 API 少大量对接工作
- 官网：https://www.xorpay.com

### 3.2 支付流程

```
用户选择套餐 → 前端调用后端创建订单
  → 后端调用虎皮椒下单 API
  → 返回支付链接/二维码
  → 前端跳转虎皮椒收银台（微信/支付宝）
  → 用户完成支付
  → 虎皮椒异步回调通知
  → 后端验签 → 更新订阅状态
  → 前端轮询/WS 通知用户支付成功
```

### 3.3 订阅周期

- **月付**：按月自动续费，到期前 3 天提醒
- **年付**：按年自动续费，到期前 7 天提醒

## 四、数据库设计

### 4.1 数据模型

```
Plan（套餐定义）
  ├── id
  ├── planId       "free" | "pro" | "enterprise"
  ├── name         "免费版" | "专业版" | "企业版"
  ├── priceMonthly  月付价格（分）
  ├── priceYearly   年付价格（分）
  ├── limits: JSON  { keywordLimit, aiLimit, hotspotDays, devices... }
  └── isActive

Subscription（用户订阅）
  ├── id
  ├── userId
  ├── planId
  ├── status       "trialing" | "active" | "past_due" | "cancelled" | "expired"
  ├── billingCycle "monthly" | "yearly"
  ├── startsAt
  ├── currentPeriodStart
  ├── currentPeriodEnd   ← 配额重置时间节点
  ├── autoRenew
  ├── cancelledAt
  └── createdAt

Payment（支付记录）
  ├── id
  ├── orderNo        商户订单号（唯一）
  ├── userId
  ├── planId
  ├── billingCycle
  ├── amount         金额（分）
  ├── currency       "CNY"
  ├── status         "pending" | "paid" | "failed" | "expired" | "refunded"
  ├── payChannel     "wechat" | "alipay"
  ├── transactionId  第三方交易号
  ├── paidAt
  ├── callbackRaw    回调原始报文（JSON）
  ├── expiresAt      订单过期时间（创建后30分钟）
  └── createdAt

Usage（配额使用）
  ├── id
  ├── userId
  ├── type           "ai_analysis" | "hotspot_view"
  ├── count          当前使用量
  ├── periodStart    当前周期开始
  ├── periodEnd      当前周期结束
  └── updatedAt

Device（多设备管理）
  ├── id
  ├── userId
  ├── deviceName
  ├── deviceType     "web" | "ios" | "android"
  ├── lastActiveAt
  └── createdAt
```

### 4.2 关联关系

```
User 1──n Subscription
User 1──n Payment
User 1──n Usage
User 1──n Device
Subscription n──1 Plan
Payment     n──1 Subscription (optional)
```

## 五、后端实现

### 5.1 API 设计

#### 用户订阅
| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/subscription` | 获取当前订阅状态 |
| GET | `/api/plans` | 获取所有套餐列表 |
| POST | `/api/subscription/cancel` | 取消自动续费 |
| POST | `/api/subscription/resume` | 恢复自动续费 |

#### 支付
| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/payments/create` | 创建支付订单 |
| GET | `/api/payments/:orderNo` | 查询订单状态 |
| POST | `/api/payments/callback/xorpay` | 虎皮椒支付回调 |
| GET | `/api/payments/history` | 支付历史 |

#### 配额
| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/usage` | 获取当前配额使用情况 |
| GET | `/api/devices` | 获取已登录设备列表 |
| DELETE | `/api/devices/:id` | 移除设备 |

### 5.2 订阅状态机

```
                    创建订单
  ───────────────────────▶ pending
                              │
                          支付成功
                              │
  trialing ◀──────────────────┤
      │                       │
      │                   active ◀──┐
      │                       │     │
  试用到期                  到期未续费  │ 续费成功
      │                       │     │
      └───────────────────── past_due ──┘
                                │
                            取消订阅
                                │
                           cancelled ──▶ expired
                                │
                            退款处理
                                │
                           refunded
```

**状态说明**：
- `trialing`：试用期（可选功能）
- `active`：正常有效
- `past_due`：欠费（宽限期 7 天，仍可使用）
- `cancelled`：已取消（到期后自动过期）
- `expired`：已过期（降级为 Free）

### 5.3 配额中间件

```
关键词添加流程：
  1. 用户提交关键词
  2. 中间件查询 User.keywordCount
  3. 对比 Plan.keywordLimit
  4. 超出 → 返回 402 + 升级引导
  5. 未超 → 正常创建

AI 分析流程：
  1. 用户发起 AI 分析请求
  2. 中间件查询 Usage.ai_analysis.count
  3. 对比 Plan.aiLimit
  4. 超出 → 返回 402 + 升级引导
  5. 未超 → 正常执行 + count++

配额重置：
  - 每月 1 日 00:00 检查所有 active 订阅
  - currentPeriodEnd < now → 重置 Usage.count = 0
  - currentPeriodEnd += 1month
```

### 5.4 定时任务

| 任务 | 频率 | 说明 |
|------|------|------|
| 重置配额 | 每月 1 日 00:00 | AI 分析次数归零 |
| 检查订阅过期 | 每小时 | past_due 满 7 天 → expired |
| 清理过期订单 | 每日 03:00 | 删除 30 分钟未支付的 pending 订单 |
| 续费提醒 | 每日 09:00 | 到期前 3/7 天发送提醒邮件 |
| 设备清理 | 每日 04:00 | 清理 30 天未活跃的设备记录 |

### 5.5 虎皮椒配置

```env
XORPAY_APP_ID=your_app_id
XORPAY_APP_KEY=your_app_key
XORPAY_MODE=sandbox   # sandbox 测试 / live 生产
XORPAY_CALLBACK_URL=https://your-domain.com/api/payments/callback/xorpay
```

## 六、前端页面

### 6.1 路由结构

```
/pricing          套餐对比页（无需登录也可访问）
/subscription     会员中心（需登录）
/billing          账单与续费（需登录）
/settings         用户设置 → 会员管理 tab
```

### 6.2 套餐对比页 `/pricing`

**布局**：
- Hero 区：产品介绍 + CTA
- 套餐卡片：Free / Pro / Enterprise 并排
- 年付切换器：月付 ↔ 年付（年付突出折扣）
- 当前套餐高亮 + 「当前方案」标签

**交互**：
- 未登录：点击「选择」→ 跳转登录 → 登录后跳转回来
- 已登录 Free：Pro/Enterprise 显示价格，点击后唤起支付
- 已登录 Pro：当前方案高亮，升级按钮 / 降级提示

### 6.3 会员中心 `/subscription`

**内容**：
- 当前方案卡片（套餐名 + 到期时间 + 自动续费状态）
- 配额使用进度
  - 监控词：已用 X / 上限 Y
  - AI 分析：已用 X / 上限 Y
- 到期提醒 banner（临期 7 天内显示）
- 操作按钮：续费 / 升级 / 取消订阅

### 6.4 账单页 `/billing`

**内容**：
- 当前订阅状态摘要
- 历史订单列表
  - 订单号 / 套餐 / 金额 / 支付方式 / 状态 / 时间
  - 支持查看详情
- 发票申请（可选 P2）

### 6.5 仪表盘配额展示

**位置**：Dashboard 顶部或侧边栏

**展示**：
```
🔥 AI 分析  42 / 500  ████████░░░░░  剩余 458 次
📡 监控词   3 / 20    ██████████░░░░░░░░░  剩余 17 个
```

## 九、注意事项

### 9.1 虎皮椒接入要点
- 测试模式（sandbox）先验证全流程，再切换生产
- 回调必须验证签名，防刷单
- 订单金额以后端为准，前端只传 planId
- 防止重复回调：记录 transactionId，已处理则忽略

### 9.2 安全要点
- 配额检查在后端，前端数据不可信
- 管理后台需 admin 角色校验
- 订单查询需校验 userId，防止越权

### 9.3 虎皮椒资料准备
- [ ] 注册虎皮椒账户（https://www.xorpay.com）
- [ ] 申请应用，获取 APP_ID 和 APP_KEY
- [ ] 配置回调地址：`https://你的域名/api/payments/callback/xorpay`
- [ ] 微信支付 + 支付宝资质审核
