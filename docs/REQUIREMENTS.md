# 📋 需求文档 - 热点监控工具

## 1. 项目背景

作为 AI 编程博主，需要：
- 第一时间获取 AI 领域热点（如大模型更新）
- 不依赖人工手动搜索
- 自动发现热点变化
- 及时收到通知

## 4. 数据模型架构

### 4.1 整体设计思路

采用**全局词库 + 用户关联**的架构设计：
- **全局词库**：所有用户共享的关键词库，新增关键词统一入库
- **用户关联**：用户选择监控哪些关键词，每个用户可独立开关
- **全局热点库**：扫描到的热点统一存储，所有用户共享同一份数据

### 4.2 数据表设计

```
┌─────────────────────────────────────────────────────────────┐
│                   KeywordLibrary (全局关键词库)               │
├─────────────────────────────────────────────────────────────┤
│  id           String  @id @default(uuid())                  │
│  text         String  @unique  // 关键词内容，全局唯一        │
│  category     String? // 分类（AI/科技/商业等）               │
│  createdAt    DateTime @default(now())                      │
│  updatedAt    DateTime @updatedAt                           │
│  userCount    Int      @default(0)  // 被多少用户监控        │
├─────────────────────────────────────────────────────────────┤
│                     UserKeyword (用户监控关系)                │
├─────────────────────────────────────────────────────────────┤
│  id           String  @id @default(uuid())                 │
│  userId       String                                    │
│  keywordId    String  // 关联到 KeywordLibrary              │
│  isActive     Boolean @default(true)  // 是否启用扫描       │
│  addedAt      DateTime @default(now())                     │
├─────────────────────────────────────────────────────────────┤
│  @@unique([userId, keywordId])  // 同一用户不重复订阅        │
└─────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────┐
│                      Hotspot (全局热点库)                    │
├─────────────────────────────────────────────────────────────┤
│  id, title, content, url, source, relevance, importance...   │
│  keywordId    String?  // 关联到 KeywordLibrary（来源关键词）│
│  scannedAt    DateTime @default(now())                      │
├─────────────────────────────────────────────────────────────┤
│  @@unique([url, source])  // 按来源URL去重                   │
└─────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────┐
│                   UserHotspotView (用户热点查看记录)          │
├─────────────────────────────────────────────────────────────┤
│  id           String  @id @default(uuid())                  │
│  userId       String                                    │
│  hotspotId    String                                    │
│  isRead       Boolean @default(false)                      │
│  viewedAt     DateTime @default(now())                    │
├─────────────────────────────────────────────────────────────┤
│  @@unique([userId, hotspotId])                              │
└─────────────────────────────────────────────────────────────┘
```

### 4.3 关键词操作流程

```
用户添加关键词:
  1. 搜索/输入关键词
  2. 检查 KeywordLibrary 是否存在
     ├─ 存在 → 直接创建 UserKeyword 关联
     └─ 不存在 → 新增到 KeywordLibrary，再创建关联
  3. KeywordLibrary.userCount +1
  4. 触发后台扫描（异步）

用户取消监控:
  1. 删除 UserKeyword 记录
  2. KeywordLibrary.userCount -1
  3. 若 userCount=0，保留关键词（其他用户可能使用）

热点扫描:
  1. 获取所有 isActive=true 的 UserKeyword
  2. 按关键词分组扫描
  3. 热点写入 Hotspot 表（已存在则跳过）
  4. WebSocket 推送给订阅该关键词的用户
```

### 4.4 用户视角的数据访问

```
用户看到的热点 = 
  该用户订阅的关键词 (UserKeyword.isActive=true)
  → 对应的热点 (Hotspot.keywordId)
  → 排除已读 (UserHotspotView.isRead=false)
```

## 6. AI 分析规格

### 5.1 OpenRouter 集成

**用途：**
1. **真假识别** - 判断内容是否为假冒/标题党
2. **热点分析** - 评估热点价值和相关性
3. **内容摘要** - 生成热点摘要

**模型选择：**
- 主模型：Claude 或 GPT-4
- 备用模型：Gemini

**Prompt 设计：**
```
你是一个热点分析专家，请分析以下内容：
1. 判断是否为真实的热点新闻（排除标题党、假新闻）
2. 评估该热点与指定领域的相关性（0-100分）
3. 评估热点的重要程度（低/中/高/紧急）
4. 生成简短摘要（50字以内）

输出 JSON 格式：
{
  "isReal": true/false,
  "relevance": 0-100,
  "importance": "low/medium/high/urgent",
  "summary": "..."
}
```

## 8. 接口设计

### 7.1 RESTful API

```
# 关键词管理
GET    /api/keywords         # 获取所有关键词
POST   /api/keywords         # 添加关键词
PUT    /api/keywords/:id     # 更新关键词
DELETE /api/keywords/:id     # 删除关键词

# 热点数据
GET    /api/hotspots         # 获取热点列表
GET    /api/hotspots/:id     # 获取热点详情
POST   /api/hotspots/search  # 手动搜索热点

# 设置
GET    /api/settings         # 获取设置
PUT    /api/settings         # 更新设置

# 通知
GET    /api/notifications    # 获取通知历史
```

### 7.2 WebSocket 事件

```
# 服务端 -> 客户端
hotspot:new      # 新热点发现
hotspot:update   # 热点更新
notification     # 通知消息

# 客户端 -> 服务端
subscribe        # 订阅关键词
unsubscribe      # 取消订阅
```
